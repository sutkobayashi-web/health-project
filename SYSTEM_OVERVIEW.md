## 1. プロジェクト概要

### 背景

運輸業界は深刻な健康課題を抱えている。長時間の座位運転、不規則な食事・睡眠、高齢化する従業員構成（平均年齢50歳、最高齢70歳）など、一般企業とは異なる固有のリスク要因が存在する。従来の産業保健施策は会社が決定し社員に通達する「トップダウン型」であり、社員の自発的な健康行動には結びつきにくかった。

### 目的

社員一人ひとりの声を出発点とし、科学的根拠（エビデンス）に基づいたテーマを全社で選び、楽しみながらチャレンジする**ボトムアップ型健康経営支援システム**を構築する。

### 対象組織の特徴

| 項目 | 内容 |
|------|------|
| 業種 | 運輸業（トラックドライバー中心） |
| 従業員平均年齢 | 約50歳（最高齢70歳） |
| ITリテラシー | 低〜中（スマートフォン操作が中心） |
| 健康リスク | 腰痛、睡眠障害、生活習慣病、メンタルヘルス |
| 従来の課題 | トップダウン施策への参加率低迷、健康行動の定着困難 |

### 3つのキーワード

- **ボトムアップ** -- 社員の声から始まる健康施策
- **全員参加** -- 投票・チャレンジ・ランキングで自然な巻き込み
- **エビデンスベース** -- 18領域の公的ガイドライン + 行動変容フレームワーク

## 2. 設計思想

### 2.1 ボトムアップ型 vs トップダウン型

本システムの根本的な設計方針は**「相手が自ら動きたくなる仕組み」**を作ることにある。

| 観点 | トップダウン型（従来） | ボトムアップ型（本システム） |
|------|------------------------|------------------------------|
| 起点 | 会社・産業医・保健師が決定 | 社員の投稿・悩み・気づき |
| テーマ選定 | 経営判断・法令対応 | 社員の声 → AI凝集 → 全社投票 |
| 参加動機 | 義務・指示 | 共感・興味・自発的エントリー |
| フィードバック | 年次健診結果の通知 | 日次KPI・ランキング・バッジ |
| 専門家関与 | 施策の企画・実行主体 | 監修・助言（年15回程度） |
| 持続性 | 施策終了で行動も終了 | サイクル継続 + AI伴走 |

### 2.2 受動から能動への転換

従来の「受け身のアプローチ」では相手の本気度が低く、コストだけがかかる。本システムは以下の転換を実現する。

- **投稿** -- 健康課題を自分の言葉で表現（受動的な健診通知からの脱却）
- **共感** -- ワンタップで他者の悩みに反応（7種の共感タイプ）
- **投票** -- テーマを自分で選ぶ（会社が決めるのではなく）
- **チャレンジ** -- 任意参加・途中離脱OK（強制しない）
- **ヘルスバディー** -- AIが楽しく話しかける（保健指導を感じさせない）

### 2.3 行動変容理論の実装

本システムは以下の学術的フレームワークを設計に組み込んでいる。

| フレームワーク | 出典 | 主な技法 | システム内での適用 |
|----------------|------|----------|-------------------|
| **EAST** | ナッジ運動編（帝京大学提供） | Easy・Attractive・Social・Timely | チャレンジ参加の簡便性、ランキングの社会性 |
| **CAN** | ナッジ食事編（帝京大学提供） | Convenient・Attractive・Normative | 食事写真AI判定、野菜チャレンジ |
| **MINDSPACE** | ナッジ禁煙編（帝京大学提供） | Messenger・Incentives・Norms・Defaults・Affect | バディーのメッセンジャー効果、QUOカードインセンティブ |
| **COM-B** | 15のカタカナ・テクニーク（帝京大学提供） | リスクアセスメント、モチベーション、ゴール&スモールステップ、モニタリング&フィードバック等15技法 | AI 7軸評価の基盤、チャレンジ設計、バディー会話 |

### 3.2 四者の役割

| 役割 | 主な責務 |
|------|----------|
| **一般社員** | 投稿 → 共感 → 投票 → エントリー → KPI記録 → ランキング参加 |
| **推進メンバー** | 投稿へのコメント → メンバー間議論 → AI評価実行 → テーマ確認 → チャレンジレビュー → 伴走 |

### 3.3 年間サイクル（約2.5ヶ月 x 4〜5回）

```

    → アンバサダー監修 → 推進メンバーレビュー → 任意参加エントリー
    → KPI記録+ランキング競争 → 成果レポート(AI+専門家所見)
```

## 4. 主要機能一覧

### 4.1 ヘルスバディー（AIパーソナルヘルスコーチ）

アプリ起動時にAIが話しかけるチャット型保健指導システム。ユーザーは「AIと楽しく話しているだけ」で、裏では組織の健康データが蓄積される。

| 機能 | 詳細 |
|------|------|
| **デイリーグリーティング** | 時間帯に応じた挨拶 + 今日は何の日 + 季節の健康トピック（1日1回キャッシュ） |
| **体調チェック** | 4段階タップ式（元気/普通/少し疲れ/しんどい）→ 深掘り選択肢（睡眠/腰肩/ストレス/食欲/目の疲れ） |
| **自由チャット** | エビデンスベースの健康相談。コーチング重視（ティーチングより質問で気づきを促す） |
| **カスタムアバター** | 名前自由設定 + アバタービルダーで外見カスタマイズ |
| **5つの性格** | gentle（やさしい）/ energy（元気）/ smart（知的）/ calm（穏やか）/ reliable（頼もしい）→ AIの口調が変化 |
| **メモ機能** | AI回答にメモボタン → 保存/解除トグル → マイページにメモ一覧 |
| **推進メンバー連携** | 「推進メンバーに相談」→ 匿名で会話内容を投稿API送信 → admin側で返信可能 |
| **チャレンジ連動** | 会話中にactiveチャレンジを自然に紹介・参加促進 |
| **話題共感** | 推進メンバーが設定した話題をバディーが能動的に問いかけ、共感・意見を収集 |
| **メッセージ配信** | 推進メンバーがバディー経由でユーザーに自然にメッセージ配信 |

**AIの行動原則**:
- 「保健指導を成功させる15の『し』と『じ』」に基づく設計
- エビデンスのない助言は絶対にしない
- 助言時は必ず出典ガイドラインを明記
- 行動変容フレームワーク（EAST / CAN / COM-B / ゴール&スモールステップ）を活用

**リッチ表示**:
- アクションカード（具体的手順提案、パープルカード表示）
- 出典カード（ガイドライン名、グリーンカード表示）

### 4.2 v2凝集型サイクル

従来の推進メンバーへの負荷集中（個別投稿評価・7軸・企画書作成）を解消し、年5本のアクションプランを全員参加型で回すフロー。

| ステップ | 内容 | 担当 |
|----------|------|------|
| 1. 声を集める | 社員が健康に関する投稿を自由に行う | 全社員 |
| 2. AIテーマ凝集 | 投稿をクラスタリングして3〜5テーマ候補を生成 | AI |
| 3. 全社員投票 | 1人2票+コメント、リアルタイム可視化（1週間） | 全社員 |
| 4. AI企画書生成 | 1位テーマからチャレンジを自動設計 | AI |
| 5. アンバサダー監修 | 保健センター専門職が3回関与（策定時・中間・最終） | 専門家 |
| 6. メンバーレビュー | AI下書きを確認・微修正のみ | 推進メンバー |
| 7. 任意参加エントリー | 途中参加/離脱OK | 全社員 |
| 8. KPI記録+ランキング | 毎日30秒、複数軸ランキング、バッジ | 全社員 |
| 9. 成果レポート | AI+専門家所見で役員報告 | AI + 専門家 |

**承認フロー（内部統制）**: テーマ確定まで5段階の承認ステップを設定

### 4.3 チャレンジKPI

チャレンジの効果を定量的に測定し、アクションプラン改善に活用する仕組み。

#### ウォーキングチャレンジ（numberタイプ）
- 会社配布の歩数計の値を毎日入力
- QUOカード褒賞: 1,000円（ヘッダー横ミニバッジ表示）

#### 野菜チャレンジ（veggie_checkタイプ）
- 食事投稿に依存しない独立KPI記録（写真を公開したくない社員への配慮）
- フロー: 「今日野菜を食べた?」→ 食事投稿から判定 or ここで写真を撮る → Gemini Vision AI判定 → スコア(0-3)で記録
- スコア基準: 野菜3種以上=3点 / 1-2種=2点 / 少量=1点 / なし=0点
- QUOカード褒賞: 500円

#### 睡眠チャレンジ（choiceタイプ）
- 就寝/起床時刻 + 睡眠の質を記録

#### チャレンジ反応収集
- バディーチャット内でチャレンジ言及時に反応ボタン表示
- 反応種別: やってみたい / 興味がある / ちょっと面倒 / もう参加中
- admin画面にバー形式の反応集計表示、AI凝集にも注入

### 4.4 食事チェック（AI栄養診断）

| 機能 | 詳細 |
|------|------|
| 食事写真分析 | Gemini Visionで画像分析 → 推定カロリー/五大栄養素+塩分/良い点/不足/ちょい足し提案 |
| 栄養バランスチャート | 5軸レーダーチャート（たんぱく質・脂質・炭水化物・ビタミン・ミネラル）をSVGで描画 ＋ 塩分は横グラデーションバーで別表示（少ない〜適量〜多い）。1食の目標摂取量に対する達成度を1〜5で可視化 |
| 週間食事レポート | 毎週月曜7:00自動実行、AI栄養アドバイザーがユーザーごとに週間栄養分析を生成・配信 |
| 食事共感 | 6種のワンタップ共感（おいしい/参考になる/同じの食べた/ヘルシー/改善したい/もっと見たい） |

### 4.5 アバターチャレンジ

- ユーザー + バディーのペア表示によるギャラリー + ランキング
- カスタムアバタービルダー（髪型・顔パーツ・カラー等を自由設定）
- 髪のツヤ線・毛の流れ線・グラデーションコントラスト強化による高品質描画

### 4.6 AI評価エンジン（7軸COM-B評価）

AIが投稿・共感データ・メンバーコメント・チャットを総合分析し、COM-Bモデル準拠の7軸で自動評価する。

| 評価軸 | 説明 |
|--------|------|
| 法令適合性 | 安全衛生法・健康増進法等への適合度 |
| リスク影響度 | 対象者の健康リスクへの影響の大きさ |
| 発生頻度 | 当該健康課題がどれだけ頻繁に報告されるか |
| 緊急度 | 即時対応の必要性 |
| 安全関連性 | 運転業務の安全への直接的影響 |
| 経営価値 | 健康経営上の戦略的価値 |
| 従業員ニーズ | 社員からの要望の強さ |

**評価入力データ**: 投稿テキスト + 部署情報 + 共感（タイプ別件数）+ メンバーコメント + メンバー議論 + 18領域エビデンス

### 4.7 AIプロンプト体系（22種）

| カテゴリ | 種数 | 主な機能 |
|----------|------|----------|
| ユーザー向けAI | 5種 | ヘルスアドバイザーチャット、初回あいさつ生成、画像分析（食事・健診）、投稿ケアコメント、健康ナレッジBot |
| 管理者向けAI | 9種 | 7軸AI評価、総合7軸評価、共感→スコア変換、AI会議シミュレーション、AIリアクション、専門家アドバイス、栄養レポート、類似投稿検索、テーマ自動凝集 |
| 企画・チャレンジ生成AI | 4種 | アクションプラン生成、3パターンアイデア、企画書リファイン、チャレンジ自動設計 |
| チャレンジKPI判定 | 4種 | 野菜スコア判定（Gemini Vision）、食事投稿からの野菜判定、チャレンジ反応収集、デイリーグリーティング生成 |

**共通ルール（全プロンプト）**:
- エビデンスのない助言は絶対にしない
- 助言時は必ず出典ガイドラインを明記
- 行動変容フレームワーク（EAST / CAN / COM-B / ゴール&スモールステップ）を活用

### 4.8 お知らせ・メッセージ配信

| 配信方式 | 内容 |
|----------|------|
| お知らせ（notice） | 全体通知・個別通知。サーバーDBで既読管理（端末間同期） |
| バディー経由メッセージ | 推進メンバーがバディーチャット内に自然にメッセージ配信 |
| 投稿者への返事 | admin画面から個別投稿者にお知らせ通知で返信 |
| 週間食事レポート | 毎週月曜自動配信 |

### 4.9 プライバシー設計

| 対策 | 詳細 |
|------|------|
| 匿名相談 | 推進メンバーへの相談時、会話内容は匿名化 |
| 野菜チャレンジ独立記録 | 写真を公開したくない社員への配慮（食事投稿不要） |
| 健診データ都度取得 | エルスケアネットワーク健診データはBox APIから都度取得、サーバー保存なし |
| 健診閲覧誓約 | 6項目の誓約チェック + アクセスログ記録 |
| 同時ログイン防止 | session_tokenによる1アカウント1端末制限 |
| バディー会話の隔離 | 会話内容は本人のみ参照可能 |
| セキュリティヘッダー | キャッシュ無効化、レートリミット |

## 7. 大学院連携と学術的位置づけ

### 7.1 連携状況

- 大学院と連携中。論文にて本システムについて言及予定（2026年3月時点）
- 本システムの設計思想と実装成果が学術的評価の対象となっている

### 7.2 思想的対立: トップダウン型産業保健 vs ボトムアップ型システム

本システムの設計と従来の産業保健学の主流アプローチには根本的な思想対立が存在する。

| 観点 | 大学側（トップダウン型） | 本システム（ボトムアップ型） |
|------|--------------------------|------------------------------|
| アプローチ | 全体主義的・組織論的 | 個人の声・共感駆動 |
| 施策決定 | 産業医・保健師・経営層が主導 | 社員の投稿 → AI凝集 → 全社投票 |
| 理論基盤 | 産業保健・労働衛生学 | 行動変容理論（EAST/COM-B/MINDSPACE） |
| 専門家の役割 | 施策の企画・実行の主体 | 監修・助言（アンバサダー） |
| AIの位置づけ | 補助ツール | テーマ凝集・企画生成・伴走の中核 |
| 成果測定 | 健診数値・受診率 | KPI日次記録 + ランキング + 行動変容 |

### 7.3 学術的新規性

- 運輸業特化のボトムアップ型健康経営システムは業界にほぼ前例がない
- 行動変容フレームワーク（EAST/COM-B/CAN/MINDSPACE）の統合的実装
- AIによるテーマ凝集・企画自動生成・パーソナルコーチングの一気通貫
- 22種のAIプロンプトによるエビデンスベースの保健指導システム

## 8. 将来構想

### 8.1 統合ドライバー管理プラットフォーム

データSIM + Android端末1台で運行管理・健康管理・安全管理を完結するプラットフォーム構想。

| システム | 機能 | 状態 |
|----------|------|------|
| SafeDrive | 運行管理（ITPデジタコ連携） | 稼働中 |
| CoWell Health | 健康経営支援（本システム） | 稼働中 |
| Safety Navi | 安全管理 | 稼働中 |

### 8.2 SaaS外販モデル（目標: 300台）

| 項目 | 詳細 |
|------|------|
| SaaS利用料 | 2,500〜3,500円/台/月 |
| 内部統制コンサル料 | 50,000〜100,000円/社/月 |
| 目標規模 | 10社（計300台）→ 月商230万円、年商2,760万円 |
| インフラ | VPS 2台（月額12,000円）で500台まで対応可能 |
| 端末 | SHARP AQUOS wish4推奨（約22,000円、技適確実、MIL規格） |
| 1台あたり原価 | 約2,600円/月（SIM+端末償却+BLE機器+保守+サーバー按分） |

### 8.3 BLEデバイス連携

| デバイス | 用途 | 想定機器 |
|----------|------|----------|
| スマートバンド | 心拍・SpO2モニタリング | Xiaomi Smart Band 8（約5,000円） |
| アルコール検知器 | 2022年義務化対応 | タニタ EA-100等（BLE対応） |
| 血糖値モニター | 糖尿病リスク者向け | Abbott FreeStyleリブレ2（14日/約7,500円） |

**段階的導入**: まず手動入力UIを先行実装 → デバイス決定後にBLE自動取得へ移行

### 8.4 ビジネスの差別化要因

- **売るのは「システム」ではなく「会社を守る内部統制の仕組み」+ それを動かすツール**
- 内部統制評価者 x IT開発 x 運送業の現場理解 x 健康経営 の4領域を兼備（業界で極めて稀）
- 中小運送業が統合システムを導入できない理由（開発力なし、IT人材なし、法令対応で精一杯）を解決
