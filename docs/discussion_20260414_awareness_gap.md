# CoWell 設計議論: 無関心層への到達と「関心の芽」の発芽

**日付:** 2026-04-14
**参加:** 小林猛（代表取締役）× Claude

---

## 1. 問題提起

CoWellのアクアリウムは「魚=自分、水槽=自分の環境」という比喩で、健康への気づきを間接的に与える設計。
しかし現状の実装は、無関心層を抱き込むには「乱暴な展開」になっている。

- 魚を元気にするために → 野菜を食べろ、運動しろ、血圧測れ
- これは結局「健康のために健康行動をしろ」と同じ構造
- 無関心層が最も嫌うのは「正しいことを言われること」

---

## 2. 正論と反論のペア分析

### 2-1. 情報提供アプローチ
- **正論:** 健康情報を与えれば関心が芽生える
- **反論:** 無関心層は健康情報を積極的に避ける。健診結果すら見ない人に情報は届かない

### 2-2. 危機感アプローチ
- **正論:** 「このままだと病気になる」と危機感を与えれば動く
- **反論:** 双曲割引により「将来の損失」より「今日の快楽」が勝つ。運送業は毎日リスクの中にいて、リスク感覚が一般人と異なる

### 2-3. 鏡（メタファー）アプローチ
- **正論:** 魚という「自分の鏡」で間接的に客観視させる
- **反論:** 鏡だと気づかなければゲームキャラ。気づいた瞬間「説教装置」に変わる。気づかせるタイミングと方法が未設計

### 2-4. 発芽の設計
- **正論:** 「関心の芽」を発芽させる仕組みを設計すべき
- **反論:** 「発芽させる」は外部から芽を植える行為=形を変えた押し付け。本当の発芽は本人が自分の中から見つけるもの

### 2-5. デジタルペットへの愛着
- **正論:** 名前をつけ、成長を見守れば愛着が湧く
- **反論:** たまごっちは数ヶ月で飽きられた。デジタルペットの愛着は消耗品。CoWellの魚は「自発的に面白いことをしない」

### 2-6. AIバディーの限界
- **正論:** 魚が話しかけてくるのでペットとは違う
- **反論:** 話しかけてくるのはAIであって魚ではない。AIの言葉に心を動かされた経験がどれだけあるか

---

## 3. 到達した仮説

### 仮説A: 関心の芽はすでに全員の中にある
- 腰が痛いドライバーは「腰が痛い」と思っている。でもそれを「健康問題」とは認識していない
- 野菜を食べない人は嫌いなのではなく、選択肢にないだけ
- 運動しない人は嫌いなのではなく、きっかけと仲間がないだけ
- **問題は「関心がない」ことではなく「健康という言葉と結びついていない」こと**

### 仮説B: 迂回路が行き止まりになっている
- アクアリウムは「健康」の壁を迂回するために設計された
- しかし食事投稿→水質変化→「野菜を食べてないから」の流れで、迂回路の先に結局「健康指導」がある
- **「健康」という言葉を使った瞬間に無関心層は離脱する**

### 仮説C: 楽しさが足りない
- 魚が元気=自分が元気 ではなく
- **魚が元気=なんか楽しい=続けたい=結果的に自分も元気だった**
- この「なんか楽しい」の部分がCoWellに不足している
- 健康は手段ではなく結果であるべき

## 4. 未解決の問い

> 「健康という言葉を使わずに、結果的に健康になる体験」とは具体的に何か？

この問いに対する答えはまだ出ていない。

## 5. 大学院連携との接続

- 大学院生の関心: **アプリ使用前と使用後でヘルスリテラシーがどう変わったか**
- 福田吉治教授のナッジ理論との接続点: 無関心層へのアプローチ手法
- CoWellはナッジの実践ツールとして位置づけられるが、現状のナッジ設計が「健康」に直結しすぎている
- 測定すべきは「健康行動の変化」ではなく「健康への関心度の変化」かもしれない

---

## 6. 今後の方針

- 4月末のローンチでは現状の仕組みで走る（完璧を目指さない）
- ローンチ後の実データを観察して「誰が・なぜ・続けたか/やめたか」を分析
- 「関心の芽」の正体を実データから探る
- 正論と反論のペア思考を継続し、安易な解決策に飛びつかない

# 第2部: 設計の変遷と「意図せぬ価値」

## 7. CoWellの「そもそも」の連鎖

| Phase | 設計意図 | そもそも |
|---|---|---|
| 悩み相談 | アプリで気軽に相談 | アプリで悩みを打ち明ける人がいるか？ |
| AIバディー | 対話から本音を引き出す | AIに本音を話す人がいるか？ |
| 熱帯魚 | 自分の投影で気づく | デジタルの魚に自分を重ねるか？ |
| 仲間の投稿 | 社会的影響で行動変容 | 他人の食事を見て自分が変わるか？ |

全て「そもそも、するか？」で止まる。
しかし理屈で答えは出ない。実装して走らせてデータで検証するしかない。

## 8. 4番目の出来事

健康に気づく出来事を3つ挙げたが、現場の声から4番目が見えた:

1. 自分の体に異変を感じた時
2. 身近な人に何かが起きた時
3. 大切なものを通じて間接的に気づいた時
4. **意図せず結果を見てしまった時**

食事の写真を撮る → 勝手に栄養分析が出る → 最初は無視 → 何回か見るうちに「俺、塩分多いな」と気づく。
健康のために行動したのではなく、**別の動機（記録・共有・好奇心）で行動した結果、健康情報に触れた**。

## 9. 設計意図と実際に起きていることのズレ

| 設計の意図 | 実際に起きていること |
|---|---|
| 健康への気づき | 献立の参考 |
| 自分の投影（魚） | まだ実感薄い |
| ボトムアップ施策 | まだ未稼働 |
| 食事診断 | **「参考になる」「役立った」の声あり** |

設計の意図と違う場所で価値が生まれている。
これは失敗ではなく、**設計者が想定していなかった到達経路**。

## 10. CoWellの出発点（小林氏の原体験）

```
健康診断の有所見率が高い
  → 会社が初診料を負担して病院に行かせた（トップダウン）
    → いやいや行くが、本気では取り組まない
      → 私（小林）が考えた健康施策を展開した
        → 全くの無意味。形だけ

結論: トップダウン型の健康施策は機能しない

転換: だったら自分たちで考えさせよう（ボトムアップ）
  → でも「考える場」がない。中小零細にはリソースもない
    → AIとアプリで場とリソースを補う = CoWell
```

## 11. 成功指標

> 「健康診断の有所見率が低下すれば、このプロジェクトは成功」（小林氏）

- **正論:** 有所見率は客観的で最も正しい指標
- **反論:** 1〜2年かかる。途中で効果なしと判断されるリスク
- **正論:** だから中間指標（利用率・投稿数・参加率）が必要
- **反論:** 中間指標はアプリの成功であって健康の成功ではない

## 12. 残された2つの問い

**問い1:** 食事投稿→食事診断→「塩分多いな」→次の食事が少し変わる——この連鎖は有所見率の低下に繋がるのか？

**問い2:** ボトムアップで社員が自ら考えた施策は、トップダウンの施策より有所見率を下げるのか？

どちらもローンチ後のデータでしか検証できない。

## 13. 現時点での到達点

- 「関心の芽」を外から植えることはできない（押し付けになる）
- 双曲割引がある限り、3つの出来事なしに健康には気づかない（極論だが真実に近い）
- しかし4番目「意図せず結果を見てしまう」は設計可能かもしれない
- 食事診断に価値を感じている声は、設計意図と違う経路での到達
- CoWellは「気づかせるツール」ではなく「気づいた瞬間の受け皿」として最も価値があるかもしれない
- ボトムアップ vs トップダウンの効果差は実データで検証するしかない
- **答えは出ていない。しかし問いの精度は上がった**

# 第3部: AIと人間の協働

## 14. 最適解の出し方

> 「人間はAIを手に入れた。絶対解は導けなくとも最適解は出せるはず」（小林氏）

AIが出せるのは「正しい答え」ではなく「問いの精度を上げること」。
正論と反論を高速で回し、人間が直感では言語化できないものを構造化する。

**最適解 = 人間の直感 × AIの構造化 × 実データの検証**

- **人間の直感:** 30年の現場経験から「トップダウンは無意味」と知っている
- **AIの構造化:** なぜ無意味なのかを双曲割引・ナッジ理論・行動経済学で言語化する
- **実データの検証:** ローンチ後に「誰が・なぜ・続けた/やめた」で仮説を検証する

CoWellはこの3つの掛け算の実験場である。
