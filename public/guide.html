Health Project v2 System Guide

このガイドについて

CoWell Health Project 2.0 の全体像をまとめたガイドページです。
プロジェクトの目的、フロー、役割分担、各機能の仕組みを分かりやすく解説します。

セクション1: プロジェクトの目的

これまでの健康施策は、会社が決めて社員に伝えるトップダウン型でした。
CoWell Health Project 2.0 では、社員の声から始まり、みんなで考え・選び・取り組む「ボトムアップ型」に転換します。

3つのキーワード
ボトムアップ 全員参加 エビデンスベース

社員一人ひとりの健康に関する悩みや気づきを出発点にし、
科学的根拠（エビデンス）に基づいたテーマを全社で選び、
楽しみながらチャレンジする仕組みです。

セクション3: 役割分担

4つの役割それぞれに明確な担当範囲があります。

👤 一般社員

  • 健康相談・食事写真を投稿
  • 他の社員の投稿に共感（7種/6種から選ぶだけ）
  • テーマ投票（2票まで）
  • チャレンジにエントリー・KPI記録
  • ランキングで競争

🔧 推進メンバー

  • 投稿一覧を確認、注目投稿にコメント
  • メンバー同士でチャット議論
  • AIが自動実行した7軸評価の結果をレビュー
  • テーマ候補を採用/除外トグルで絞り込み
  • カスタムプラン（D以降）を追加（タイトル・説明・KPI付き）
  • 全プラン（AI案+メンバー案）に7種共感ボタンで評価
  • チャレンジ内容をレビュー
  • 実行中の伴走（声かけ・励まし）

🩺 保健師（アンバサダー）

  • 採用テーマのみ表示（除外テーマは非表示）
  • AIプラン+メンバープラン+共感データ+議論を一覧で確認
  • プランごとに専門的助言を記入（必須）
  • チャレンジ中間チェック
  • 最終レポートへの所見
  • /ambassador 画面からアクセス

👔 役員

  • テーマの最終承認（必須・Exec権限）
  • 承認者名が記録される
  • 差戻し可能

セクション5: ヘルスバディー

ヘルスバディーとは

アプリを開くと最初に表示されるAI健康相棒です。チャットがメインビューとなっており、毎日あなたに話しかけ、体調を確認し、健康に関する悩みを自然な会話の中で引き出します。

初回会話時には、バディーがアプリの仕組みを紹介し、カスタマイズを促します。バディーの名前・見た目・性格を設定して、自分だけの健康パートナーを作れます。

1
毎日の体調チェック
アプリを開くとバディーが「今日の調子はどう？」と話しかけます。
😊元気 / 😐普通 / 😔少し疲れ / 😩しんどい の4択をタップするだけ。
2
悩みの深掘り
体調に応じて「何か気になることは？」と選択肢を提示します。
💤睡眠 / 🦴腰・肩 / 😣ストレス / 🍽️食欲 / 👁️目の疲れ など。
「💬 自分の言葉で話す」で自由に相談することもできます。
3
エビデンスに基づくアドバイス
AIが厚生労働省のガイドラインや最新の研究に基づいてアドバイスします。
具体的なアクション（🎯やってみよう）と出典（📚）がカード形式で表示されます。
4
推進メンバーへの橋渡し
AIだけでは解決できない悩み（職場の人間関係、医療相談など）は「🛡️推進メンバーに相談」で匿名で届けられます。
推進メンバーがお知らせ通知で返事をします。
5
チャレンジとの連動
会話の中で現在進行中のチャレンジに関連する話題が出ると、AIが自然に参加を促します。

バディーのカスタマイズ

マイページの「バディーを変更する」ボタンから、いつでもカスタマイズできます。

項目内容
見た目アバタービルダーで髪型・目・口・アクセサリーなどを自由にカスタマイズ
名前好きな名前を付けられます（例：ハルちゃん、ケンタ先生）
性格5種類から選択。AIの口調が変わります
性格特徴口調の例
🌸 やさしい温かく寄り添う「大丈夫ですよ」「一緒にがんばりましょうね」
⚡ 元気いっぱいテンション高めで励ます「やったね！」「いいね！すごい！」
📋 しっかり者明確で的確なアドバイス「ここがポイントです」「具体的には〜」
😄 おもしろユーモアで楽しくダジャレや例え話を交えた親しみやすい口調
🍃 おだやかゆったり癒し系「今の自分を大切に」マインドフルネス的な声がけ

AIの行動原則（保健指導15の極意）

バディーは「保健指導を成功させる15の『し』と『じ』」に基づいて設計されています。

原則AIでの実装
準備を怠らず食事・体調・健診データを会話前に自動読み込み
信頼関係を築く過去の会話を覚えている。名前で呼ぶ
柔軟に対応ユーザーの性格設定に合わせて口調を変える
しゃべりすぎない3〜4文以内で簡潔に。選択肢を提示
指示にならない「〜してください」ではなく「〜してみない？」
自分で考えてもらう選択肢を出して本人に選ばせる
じっくり、あせらず小さな変化を褒める。失敗を責めない
自己完結しない深刻な悩みは推進メンバーへ橋渡し

裏側の仕組み

仕組み内容
テーマ織り込み推進メンバーが設定したテーマをAI会話に自然に織り込む
食事データ連動食事投稿から生活状態を読み取り、自然な気づきを促す
匿名の声の吸い上げ会話データを匿名でテーマ凝集分析に投入→v2サイクルに接続
異常検知体調不良が続く場合、推進メンバーへアラート
チャレンジ連動進行中のチャレンジをDBから都度取得し、関連する話題で自然に紹介
サーバー同期バディーデータはサーバーDBに保存。端末を変えても同じバディー

プライバシーと情報の範囲

バディーとの会話はあなただけのプライベート空間です。安心して健康相談ができます。

情報の流れ内容
チャット内容あなたとバディーの間だけ。管理者・推進メンバー・他のユーザーは一切閲覧できません
管理者 → あなた推進メンバーからバディー経由でお知らせや話題共感の問いかけを届けることができます（一方向の配信のみ）
あなた → 管理者「🛡️推進メンバーに相談」を自分の意思で選んだ時のみ、匿名で相談内容が届きます
評価対象チャット内容は管理画面の評価には使用されません。評価対象は投稿（声）のみです

チャット管理

機能内容
履歴クリアバディーヘッダー右のボタンで全チャット履歴を削除できます。新しい挨拶から再スタートします
メモ機能バディーの回答にある「メモ」ボタンで健康アドバイスを保存。マイページの「バディーメモ」で確認・削除できます

セクション5-2: バディー経由メッセージ配信

推進メンバーがバディーを通じてユーザーにメッセージを届ける機能です。お知らせとは別に、バディーチャット内に自然にメッセージが表示されます。

送信方法（admin画面）

  • 個別送信: インボックスで投稿を開く → 返事を入力 → 青い「バディー経由」ボタンをクリック
  • 全体配信:「全員へお知らせ配信」モーダル →「ヘルスバディー経由で届ける」にチェック → 配信

ユーザー側の表示

  • バディーチャット画面に表示（お知らせ一覧には出ません）
  • 個別メッセージ: 「💌 ○○さんからのメッセージ」
  • 全体配信: 「🛡️ 推進メンバーからのお知らせ」
  • 30秒ごとに新着を自動チェック（リロード不要）
  • PWAを閉じて再度開いた時にも自動チェック

仕組み

  • メッセージはDB管理で既読追跡（一度表示されたメッセージは再表示されない）
  • お知らせAPIとバディーAPIは完全に分離
  • バディーチャットの会話履歴にも保存され、同日の再訪問時に履歴として表示
セクション5-3: 話題共感（バディー経由の声集め）

推進メンバーが話題（テーマ）を設定すると、バディーがユーザーに自然に問いかけ、共感や意見を収集する機能です。従来の受け身な共感収集を、バディーから能動的に問いかける形に変えることで、参加率の向上を狙います。

推進メンバーの操作（admin画面）

  1. サイドバーの「話題共感」タブを開く
  2. 話題テーマを入力（例：「階段を使う習慣、どう思いますか？」）
  3. 選択肢を編集（デフォルト4つ、最大6つまで追加可能）
  4. 「話題を配信」をクリック → 既存の配信中話題は自動終了

ユーザー側の体験

  1. バディーチャットを開くと「📢 今週の話題です！」と問いかけ
  2. 選択肢ボタンをタップして回答
  3. 任意でコメントを追加（スキップ可）
  4. 回答後、通常の体調チェックに移行

※ 各ユーザーは1つの話題に1回のみ回答できます。回答済みの話題は再表示されません。

※ ページを開いたままでも30秒ごとに新しい話題をチェックします。adminが配信したら自動で問いかけが始まります。

集計の確認（admin画面）

  • 話題カードに選択肢ごとの回答数がグラフ表示
  • 回答率（回答者数 / 全ユーザー数）を表示
  • 「詳細・コメント」ボタンで、回答者名・部署・コメント一覧を確認
  • 話題は「終了」で配信停止、「削除」で回答データごと完全削除

v2スキームとの連携

テーマ凝集 → 全社投票 → チャレンジ の流れに、バディー経由の「空気感の汲み取り」を加えることで、投票前の温度感を測定できます。話題共感の結果をテーマ選定の参考材料として活用してください。

管理画面（推進メンバー向け）のチャット通知

セクション8: 投稿新着バッジ

自分の投稿に推進メンバーからのコメントがあると、投稿カードに「新着N件」バッジが表示されます。

1
推進メンバーが投稿にチャットコメントを追加
2
ユーザーの画面で60秒ポーリングにより未読を検知
3
自分の投稿カードに「新着N件」バッジが表示
4
バッジをタップ or マイページで投稿を開くと既読化→バッジ消去
セクション9: AI 7軸評価

7つの軸でテーマ候補を自動評価します。

軸評価内容
法令 (legal)法的義務への該当度
リスク (risk)放置時の健康リスク
頻度 (freq)組織内での普遍性

ニーズ (needs)行動変容への寄与
入力ソース: 共感データ + メンバーコメント + チャット議論 + エビデンス18領域の公的ガイドライン

凝集前の自動7軸評価

AIテーマ凝集の実行時、まだ7軸評価が済んでいない投稿は自動的に一括評価されます。
管理者が個別に評価ボタンを押す必要はありません。

凝集時のフロー
1. 凝集ボタン 2. 未評価を自動7軸評価 3. 評価レポート表示 4. 確認後に凝集実行

7軸評価レポート

凝集前に表示されるレポートでは、以下の情報を確認できます。

項目内容
スコア順一覧全投稿を合計スコア（/35）の高い順に表示。色分け（赤=25以上、橙=18以上、緑=18未満）で重要度が一目瞭然
7軸スコア各投稿の法令・危険度・頻度・緊急・安全・価値・ニーズをバッジで表示
評価根拠AIがそのスコアを付けた理由を2〜3文で表示。共感データやメンバー議論の内容が反映
参照ガイドラインスコアの根拠となった厚労省等の公的ガイドライン名
NEWバッジ今回の凝集で新たに評価された投稿を識別
注意: 共感データやメンバー議論がない段階で評価すると、頻度(freq)やニーズ(needs)のスコアが低く出る傾向があります。十分な共感・議論が集まってから凝集を実行すると、より精度の高いテーマ生成が期待できます。
セクション10: AI凝集の改善

AIテーマ凝集の安定性と効率性を大幅に改善しました。

改善項目内容
5テーマ+プラン案生成AIが5つのテーマ候補を生成し、各テーマにA/B/Cの3つのアクションプラン案（タイトル・説明・KPI付き）を自動提案
メンバーカスタムプラン推進メンバーがD以降のプランを自由に追加可能（タイトル・説明・KPI）。AI案と同列に並ぶ
7種共感ボタン全プラン（AI案+メンバー案）に対し7種の共感ボタンで評価: 👍賛成 / 🔥最優先 / 💡効果的 / 👥巻き込める / 🤔要議論 / ⚠️難しい / ❌見送り
採用/除外トグルメンバーがテーマを採用・除外に切り替えて候補を絞り込み。保健師画面には採用テーマのみ表示

バックグラウンド処理7軸一括評価をバックグラウンドジョブ化。5秒ポーリングで進捗表示（504タイムアウト回避）

セクション10-2: テーマディスカッション

テーマ候補段階から推進メンバーがプラン案の評価・追加・議論を行い、合意形成を図る機能です。

機能概要

項目内容
対象テーマ候補ごとに個別のチャットスレッド + プラン案一覧（候補段階から利用可能）
参加者推進メンバー全員（管理画面からアクセス）
タイミングAI凝集後。採用/除外トグルで絞り込み → プラン評価・追加 → 保健師助言・役員承認の前
目的AIプラン（A/B/C）+メンバー追加プラン（D〜）に共感ボタンで評価し、ディスカッションで合意形成

7種共感ボタン

ボタン意味
👍 賛成このプランに賛成する
🔥 最優先最も優先して取り組むべき
💡 効果的効果が高いと思う
👥 巻き込める多くのメンバーを巻き込めそう
🤔 要議論もう少し議論が必要
⚠️ 難しい実現が難しいと思う
❌ 見送り今回は見送るべき

使い方

  1. 管理画面でAI凝集を実行し、5テーマ候補（各A/B/Cプラン付き）を生成
  2. テーマを採用/除外トグルで絞り込み（除外テーマは保健師画面に表示されない）
  3. 各プラン（AI案A/B/C）に7種共感ボタンで評価
  4. 必要に応じてカスタムプラン（D以降）を追加（タイトル・説明・KPI）
  5. 各テーマ候補カードの「ディスカッション」ボタンで議論
  6. チャット画面でメンバー名入りのメッセージを投稿し合意形成
  7. 精査が完了したら保健師プラン別助言 → 役員承認へ進行
v2サイクルでの位置づけ
AI凝集（5候補+A/B/Cプラン） → 採用/除外トグル → メンバープラン追加（D〜） → 7種共感評価 → ディスカッション → 保健師プラン別助言 → 役員承認 → 全社投票
共感データ・議論の内容は次回以降のAI凝集にも反映され、テーマとプランの質が継続的に向上します。
保健師（アンバサダー）画面
採用テーマのみが表示され、各テーマのAIプラン・メンバー追加プラン・共感データ集計・メンバー議論を一覧で確認。
保健師はプランごとに専門的助言を記入できます（全体コメントではなくプラン単位）。

セクション19: セキュリティ対策

本システムは外部アクセス（大学関係者・保健師等）を想定し、以下のセキュリティ対策を実装しています。

対策内容
HTTPS強制全通信をSSL/TLS（Let's Encrypt + Cloudflare）で暗号化。HTTP接続は自動的にHTTPSへリダイレクト
CORS制限APIへのアクセスを health.biz-terrace.org からのリクエストのみに限定。外部サイトからの不正呼び出しを防止
CSPヘッダーContent Security Policy により、XSS（クロスサイトスクリプティング）やクリックジャッキング攻撃を防止
レート制限API全体: 15分あたり500リクエストまで。認証系（ログイン・登録）: 15分あたり50回まで。ブルートフォース攻撃を防止
JWT認証ログイン後に発行されるトークンで本人確認。有効期限24時間。秘密鍵は環境変数で厳格管理
管理API認証全ての管理操作（承認・評価・議論・バックアップ等）に管理者トークン認証が必須
パスワード保護bcryptハッシュで保存。平文パスワードはサーバーに保存されません
パスワード変更生年月日による本人確認が必須。旧パスワードなしでも変更可能（忘れた場合の救済）
ファイルアップロード制限画像タイプ検証（JPEG/PNG/WebPのみ）、5MB以下に制限、ランダムファイル名で推測不能化
リクエストサイズ制限リクエストボディを10MBに制限。大量データ送信によるDoS攻撃を防止
自動バックアップ毎日深夜2時にSQLiteデータベースをBox Cloudへ自動バックアップ
運用上の注意
・管理者パスワードは定期的に変更してください
・不審なアクセスがあった場合はサーバーログを確認してください
・JWTトークンの有効期限は24時間です。長時間操作しない場合は再ログインが必要です
セクション20: 同時ログイン防止

1アカウントにつき1端末のみログイン可能です。別の端末でログインすると、前の端末のセッションは自動的に無効化されます。

仕組み内容
セッションIDログイン時にランダムなセッションIDを発行し、DBに保存
検証API呼び出しごとにJWT内のセッションIDとDB上のIDを照合
無効化新ログインでDBのセッションIDが更新され、旧端末のIDが不一致に
通知旧端末で「別の端末でログインされました。再ログインしてください。」と表示
セクション21: よくある質問（FAQ）
Q. 投稿は匿名ですか？
ニックネームで表示されます。実名は管理者のみ確認可能です。
Q. 共感は何回できますか？
1つの投稿につき1回です。種別を選んでタップするだけで完了します。
Q. チャレンジは強制ですか？
任意参加です。途中参加・途中離脱もOKです。
Q. 保健師はどこからアクセスしますか？
/ambassador のアンバサダー画面からアクセスします。
Q. 役員承認は誰ができますか？
メンバー管理でExec権限が設定されたアカウントのみ承認可能です。
