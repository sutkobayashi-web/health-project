Health Project v2 System Guide

このガイドについて

健康プロジェクトv2の全体像をまとめたガイドページです。
プロジェクトの目的、フロー、役割分担、各機能の仕組みを分かりやすく解説します。

セクション1: プロジェクトの目的

これまでの健康施策は、会社が決めて社員に伝えるトップダウン型でした。
健康プロジェクトv2では、社員の声から始まり、みんなで考え・選び・取り組む「ボトムアップ型」に転換します。

3つのキーワード
ボトムアップ 全員参加 エビデンスベース

社員一人ひとりの健康に関する悩みや気づきを出発点にし、
科学的根拠（エビデンス）に基づいたテーマを全社で選び、
楽しみながらチャレンジする仕組みです。

セクション3: 役割分担

4つの役割それぞれに明確な担当範囲があります。

👤 一般社員

  • 健康相談・食事写真を投稿
  • 他の社員の投稿に共感（7種/6種から選ぶだけ）
  • テーマ投票（2票まで）
  • チャレンジにエントリー・KPI記録
  • ランキングで競争

🔧 推進メンバー

  • 投稿一覧を確認、注目投稿にコメント
  • メンバー同士でチャット議論
  • AI7軸評価を実行
  • テーマ候補を精査・編集
  • チャレンジ内容をレビュー
  • 実行中の伴走（声かけ・励まし）

🩺 保健師（アンバサダー）

  • テーマ候補への専門的助言（必須）
  • チャレンジ中間チェック
  • 最終レポートへの所見
  • /ambassador 画面からアクセス

👔 役員

  • テーマの最終承認（必須・Exec権限）
  • 承認者名が記録される
  • 差戻し可能

法令 (legal)法的義務への該当度

入力ソース: 共感データ + メンバーコメント + チャット議論 + エビデンス18領域の公的ガイドライン

凝集前の自動7軸評価

AIテーマ凝集の実行時、まだ7軸評価が済んでいない投稿は自動的に一括評価されます。
管理者が個別に評価ボタンを押す必要はありません。

凝集時のフロー
1. 凝集ボタン 2. 未評価を自動7軸評価 3. 評価レポート表示 4. 確認後に凝集実行

7軸評価レポート

凝集前に表示されるレポートでは、以下の情報を確認できます。

項目内容
スコア順一覧全投稿を合計スコア（/35）の高い順に表示。色分け（赤=25以上、橙=18以上、緑=18未満）で重要度が一目瞭然
7軸スコア各投稿の法令・危険度・頻度・緊急・安全・価値・ニーズをバッジで表示
評価根拠AIがそのスコアを付けた理由を2〜3文で表示。共感データやメンバー議論の内容が反映
参照ガイドラインスコアの根拠となった厚労省等の公的ガイドライン名
NEWバッジ今回の凝集で新たに評価された投稿を識別
注意: 共感データやメンバー議論がない段階で評価すると、頻度(freq)やニーズ(needs)のスコアが低く出る傾向があります。十分な共感・議論が集まってから凝集を実行すると、より精度の高いテーマ生成が期待できます。

推進メンバーのチャット内容は投稿した本人だけが閲覧できます。他の一般ユーザーには表示されません。

セクション11: よくある質問（FAQ）
Q. 投稿は匿名ですか？
ニックネームで表示されます。実名は管理者のみ確認可能です。
Q. 共感は何回できますか？
1つの投稿につき1回です。種別を選んでタップするだけで完了します。
Q. チャレンジは強制ですか？
任意参加です。途中参加・途中離脱もOKです。
Q. 保健師はどこからアクセスしますか？
/ambassador のアンバサダー画面からアクセスします。
Q. 役員承認は誰ができますか？
メンバー管理でExec権限が設定されたアカウントのみ承認可能です。
セクション12: AIプロンプト一覧

本システムでは22種のAIプロンプトを使用しています。
テキスト処理にはGroq（Llama 3.3 70B）、画像分析にはGemini Visionを使用。
全プロンプトは18領域の公的ガイドラインをエビデンス基盤として参照しています。

共通ルール（全プロンプト）
・エビデンスのない助言は絶対にしない
・マークダウン記法（**太字**や###見出し）は使用禁止。強調は【】で囲む
・助言時は必ず「📚 出典:」として根拠ガイドラインを明記
・行動変容フレームワーク: EAST / CAN / COM-B / ゴール&スモールステップを活用

ユーザー向けAI（5種）

機能モデル概要
AIヘルスアドバイザー（チャット）Groqエビデンスに基づく健康相談チャット。コーチング重視、ティーチングより質問で気づきを促す
初回あいさつ生成Groq新規チャット開始時の自己紹介と健康に関するオープンな質問
画像分析（食事・健診）Gemini Vision + Groq食事写真の栄養分析（推定カロリー/PFC/良い点/不足/ちょい足し提案）。健診結果の数値解説
投稿ケアコメント生成Groq健康相談投稿に対するAIからの温かいケアコメント + 7軸自動評価スコア
健康ナレッジBotGroq健康経営の専門知識を平易な言葉で解説。ナッジ/EAST/CAN等のフレームワークも教育

管理者向けAI（9種）

機能モデル概要
7軸AI評価Groq投稿をCOM-Bモデル準拠の7軸（法令/リスク/頻度/緊急/安全/価値/ニーズ）で1-5点評価
総合7軸評価Groq共感データ + メンバーコメント + 議論を統合した多角的7軸評価
共感→スコア変換Groqハートリアクションの種別・件数から7軸スコアを自動算出
AI会議シミュレーションGroq11人のペルソナ（20代ドライバー/50代役員/保健師等）による議論を自動生成
AIリアクションGroq人間のコメントに対するAI会議メンバーの反応を1-2件生成
専門家アドバイスGroq健康経営コンサルタントとしての専門的助言
栄養レポート生成Groqユーザーの食事履歴を分析し、傾向/良い点/改善点/目標のレポートを生成
類似投稿検索Groqテーマが似ている投稿をAIが分析してグルーピング
テーマ自動凝集Groq全投稿から3-5個の健康テーマを自動クラスタリング。共感データ・評価スコアを重み付け

企画・チャレンジ生成AI（4種）

機能モデル概要
アクションプラン生成Groq現状分析/経営根拠/実行計画/KPI指標を含む企画書をMarkdownで自動生成
3パターンアイデアGroq小さく始める案/ゲーミフィケーション案/根本解決案の3パターンを提案
企画書リファインGroqフィードバックを反映して企画書を改善。セクションごとの修正意図も出力
チャレンジ自動設計Groq30日間の任意参加プログラムを自動設計。EAST/COM-Bフレームワーク準拠、KPI定義、バッジシステム（初記録/7日連続/皆勤賞等）を含む
全プロンプトは厚労省ガイドライン・日本糖尿病学会・日本高血圧学会等の18領域の公的エビデンスを共通基盤として参照しています。 詳細は 学習ページ をご覧ください。
セクション13: セキュリティ対策

本システムは外部アクセス（大学関係者・保健師等）を想定し、以下のセキュリティ対策を実装しています。

対策内容
HTTPS強制全通信をSSL/TLS（Let's Encrypt + Cloudflare）で暗号化。HTTP接続は自動的にHTTPSへリダイレクト
CORS制限APIへのアクセスを health.biz-terrace.org からのリクエストのみに限定。外部サイトからの不正呼び出しを防止
CSPヘッダーContent Security Policy により、XSS（クロスサイトスクリプティング）やクリックジャッキング攻撃を防止
レート制限API全体: 15分あたり500リクエストまで。認証系（ログイン・登録）: 15分あたり50回まで。ブルートフォース攻撃を防止
JWT認証ログイン後に発行されるトークンで本人確認。有効期限24時間。秘密鍵は環境変数で厳格管理
管理API認証全ての管理操作（承認・評価・議論・バックアップ等）に管理者トークン認証が必須
パスワード保護bcryptハッシュで保存。平文パスワードはサーバーに保存されません
パスワード変更生年月日による本人確認が必須。旧パスワードなしでも変更可能（忘れた場合の救済）
ファイルアップロード制限画像タイプ検証（JPEG/PNG/WebPのみ）、5MB以下に制限、ランダムファイル名で推測不能化
リクエストサイズ制限リクエストボディを10MBに制限。大量データ送信によるDoS攻撃を防止
自動バックアップ毎日深夜2時にSQLiteデータベースをBox Cloudへ自動バックアップ
運用上の注意
・管理者パスワードは定期的に変更してください
・不審なアクセスがあった場合はサーバーログを確認してください
・JWTトークンの有効期限は24時間です。長時間操作しない場合は再ログインが必要です
